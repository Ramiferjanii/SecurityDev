# Hostname Configuration Guide

## When You Need to Configure Hostname

### 1. SendGrid Domain Verification

If you're verifying a domain in SendGrid for sending emails:

**Hostname to use:**
- For CNAME record: `mail.yourdomain.com` or `email.yourdomain.com`
- For SPF/DKIM records: Use the exact hostnames provided by SendGrid (e.g., `s1._domainkey.yourdomain.com`)

**Steps:**
1. Go to SendGrid → Settings → Sender Authentication → Domain Authentication
2. Add your domain (e.g., `yourdomain.com`)
3. SendGrid will provide specific hostnames for DNS records
4. Add those DNS records to your domain's DNS settings

**Example DNS Records:**
```
Type: CNAME
Host: mail.yourdomain.com
Value: (provided by SendGrid)

Type: CNAME  
Host: s1._domainkey.yourdomain.com
Value: (provided by SendGrid)
```

### 2. Appwrite Self-Hosted Instance

If you're self-hosting Appwrite:

**Hostname to use:**
- `https://appwrite.yourdomain.com` or `https://api.yourdomain.com`

**In `.env.local`:**
```env
NEXT_PUBLIC_APPWRITE_ENDPOINT=https://appwrite.yourdomain.com/v1
```

**For Appwrite Cloud (default):**
```env
NEXT_PUBLIC_APPWRITE_ENDPOINT=https://cloud.appwrite.io/v1
```
*(No custom hostname needed)*

### 3. VAPI AI Webhooks

If setting up webhooks in VAPI AI:

**Development:**
- Use ngrok: `https://your-ngrok-url.ngrok.io`
- Or localhost for testing: `http://localhost:3000`

**Production:**
- Your deployed domain: `https://yourdomain.com`

**In VAPI AI Dashboard:**
- Go to Webhooks settings
- Add your hostname/URL for callbacks

### 4. Appwrite Functions

If configuring Appwrite Functions:

**Hostname/URL:**
- Function URL is automatically generated by Appwrite
- No manual hostname configuration needed
- Access via Appwrite Console → Functions → Your Function → Settings

### 5. Production Deployment

For production deployment (Vercel, etc.):

**Hostname:**
- Your deployed domain: `https://yourdomain.com`
- Or Vercel URL: `https://your-app.vercel.app`

**Update environment variables:**
- Ensure `NEXT_PUBLIC_APPWRITE_ENDPOINT` points to your Appwrite instance
- Update any webhook URLs to use production domain

## Quick Reference

## Common Issues

**"Hostname not found"**
- Check DNS propagation (can take up to 48 hours)
- Verify DNS records are correct
- Use `nslookup` or `dig` to check DNS

**"Invalid hostname"**
- Ensure no trailing slashes
- Use full URL with protocol (https://)
- Check for typos

**"Hostname verification failed"**
- For SendGrid: Wait for DNS propagation
- For Appwrite: Check SSL certificate is valid
- For webhooks: Ensure URL is publicly accessible
